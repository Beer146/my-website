My Home Lab Project

My home lab is my personal sandbox for experimenting with DevOps workflows and cloud technologies. I started it as a way to apply what I was learning at Babbage Insight and IntelNeuron LLC to a more controlled, hands-on environment. Using an AWS EC2 instance and Docker, I created a simulated infrastructure where I can deploy and test services continuously.

I configured OpenTofu (Terraform fork) scripts to provision EC2 instances and networking resources automatically. Once my infrastructure was up, I containerized several lightweight applications and orchestrated them with Docker Compose. To automate builds and deployments, I set up a GitHub Actions pipeline that SSHs into my EC2 instance, pulls the latest code, and rebuilds Docker containers on every push.

This project also includes AWS monitoring through CloudWatch and cost tracking, helping me understand the practical challenges of running services efficiently. Ultimately, the home lab gives me the flexibility to test new tools, break things safely, and deepen my understanding of scalable infrastructure automation.

Tools and Technologies Used

  • AWS EC2 (Ubuntu Server)
  • Docker & Docker Compose
  • OpenTofu (Terraform-compatible IaC)
  • GitHub Actions for CI/CD
  • CloudWatch & AWS Billing Dashboard
